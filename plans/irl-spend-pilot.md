# IRL Spend Pilot

## Overview

This pilot lets users turn existing in-app points into real spending power at a live event.

Instead of asking users to fund USDC themselves, IRL will allow each eligible user to convert points into up to **$5 USDC on Base** one time during the experience. That USDC can then be used in a simple QR-based spend flow inside the IRL app.

The goal is to test whether IRL can make digital rewards feel immediately useful in the real world.

## Objective

Prove that IRL can deliver a simple, understandable in-person spend experience built on top of value users already have in the app.

This pilot is meant to answer a straightforward question:

**Can IRL turn existing points into a live event payment experience that feels seamless for users and manageable for operators?**

## How it works

1. Users start with points already in the IRL app.
2. IRL allows each user to convert points into USDC on Base one time, up to **$5**.
3. At the event, staff presents a QR code for a purchase experience, such as a drink ticket.
4. The user scans and confirms the transaction in the IRL app.
5. The transaction is completed using the converted USDC.
6. Operators can monitor usage and treasury balance throughout the event.

## Pilot rules

- Each user can convert points **one time only**
- Maximum conversion is **$5 per user**
- The points-to-USDC conversion rate is **fixed for the pilot**
- The pilot uses **one wallet with a fixed USDC balance**
- The experience is limited to **one event** and **one spend flow**

## Treasury model

The pilot is backed by a wallet funded with a fixed amount of USDC.

That wallet exists to support the one-time conversion from points to USDC. The size of the treasury should be based on the maximum possible exposure.

Example:

- 100 users
- $5 maximum conversion each
- minimum treasury exposure: **$500 USDC**

This model keeps the pilot simple and makes it easy to track usage, remaining balance, and overall demand.

## Why this approach

This structure is intentionally narrow.

It removes the need for users to bring their own stablecoins, reduces onboarding friction, and keeps the entire experience inside IRL. It also gives the team a clean way to test whether existing points can become a meaningful bridge between digital engagement and real-world spending.

## Risks and assumptions

This pilot assumes:

- users understand that points now have real spending value
- a one-time $5 conversion is enough to make the experience feel compelling
- treasury limits are easy to communicate if demand is higher than expected
- the conversion step feels simple rather than confusing

The main risk is that users may not immediately understand the relationship between points, USDC, and event spending. The UX and messaging will need to make that obvious.

## Open questions

- What exact conversion rate will be used for the pilot?
- Which users are eligible?
- When does conversion become available?
- What happens if treasury funds run out?
- Does unused USDC remain with the user after the event, or is it constrained to the experience?

## Success criteria

The pilot is successful if:

- users understand the value of their points in this experience
- users can complete the conversion and spend flow without confusion
- staff can operate the experience without special hardware
- operators can clearly track usage and treasury exposure during the event

## Summary

This pilot tests a simple idea:

**users already have points, and IRL lets them turn a small amount of those points into real event spending power.**

If successful, it shows that IRL can make rewards immediately useful in the real world without requiring users to fund crypto themselves.
